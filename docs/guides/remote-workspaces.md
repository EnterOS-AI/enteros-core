# External workspaces

An external workspace runs outside platform-managed compute while retaining a
Molecule workspace identity, authenticated registration, heartbeat, discovery,
and A2A delivery.

The supported setup is generated by the target tenant. In Canvas, create an
external/remote workspace and run the returned runtime-specific connection
material on the external host. Do not copy package names, version pins, token
storage paths, or registration commands from an older blog post: the handler in
the running server generates the current contract.

Start with:

- [External Workspace Quickstart](./external-workspace-quickstart.md) for the
  operator flow;
- [External Agent Registration](./external-agent-registration.md) for
  credentials, registration, secrets, and E2E checks;
- [Remote Workspaces FAQ](./remote-workspaces-faq.md) for networking and
  troubleshooting.

The platform does not manage the lifecycle or upgrades of an external process.
Use the server-generated workspace credential, verify registration and
heartbeat, then send a real Canvas message and confirm the reply returns before
considering the integration ready.
